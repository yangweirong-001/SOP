# DESIGN.md

## 产品气质与意象

**意象锚点：物流分拣中心的指挥控制室**——大面积冷静的浅灰底色让人联想到打磨光洁的水磨石地坪，蓝-靛渐变如同月台调度屏上跳动的光纤数据线，琥珀色的判断节点像传送带上闪烁的提示灯。整套界面应该传递「井然、可控、可追溯」三个词，既不是消费级 SaaS 的轻盈，也不是工业 ERP 的笨重，而是一种被反复演练过的「专业人士工作台」气场。

## 视觉策略

- 大背景：slate-50 / 微噪点，营造「可承重」的工作台感
- 卡片：纯白 + 细边（slate-200）+ 微阴影；hover 时上浮 2px 并加深阴影
- 主色块（蓝-靛渐变）仅用于：品牌 logo、主操作按钮、流程连接线、AI Agent 触发器，作为视觉焦点
- 判断节点用 amber 色系，与 action 步骤的 blue 形成色相反差，让流程节奏一眼可读

## 配色方案

| 角色 | 取值 | 意象来源 |
| --- | --- | --- |
| 主色 Brand Blue | `#2563eb` (blue-600) → `#4f46e5` (indigo-600) 渐变 | 月台调度屏的数据线 |
| 操作步骤底色 | `#eff6ff` (blue-50) + `#dbeafe` 边 | 流程图标准节点 |
| 判断节点底色 | `#fffbeb` (amber-50) + `#fde68a` 边 | 传送带上的警示黄 |
| 风险/异常 | `#fef2f2` (red-50) + `#fecaca` 边 | 警戒胶带 |
| 成功/通过 | `#ecfdf5` (emerald-50) | 验收合格章 |
| 背景画布 | `#f8fafc` (slate-50) | 打磨光洁的水磨石 |
| 主文本 | `#0f172a` (slate-900) | 重墨签字 |
| 次级文本 | `#64748b` (slate-500) | 表单说明 |

## 字体排版

- 字体族：`Inter` + `Noto Sans SC`（思源黑体子集），中文优先，CN 域加载
- 标题：font-bold，紧凑 leading（1.2）
- 正文：font-normal，舒展 leading（1.6），保证 SOP 长描述的可读性
- 数字 / SKU / 时间：font-medium，tabular-nums，方便对齐
- 卡片内层级：H3（步骤名）→ p（描述）→ meta chips（角色/工具/耗时）

## 动效与交互

- 缓动曲线：统一使用 `cubic-bezier(0.4, 0, 0.2, 1)`（ease-out 类），节奏干练
- 步骤卡片 hover：translateY(-2px) + shadow 加深，持续 300ms
- AI Agent 触发按钮：常驻脉动光环（pulse-ring），暗示 AI 在线
- 流式回复：打字机式增量渲染 + 三点 typing-indicator
- 步骤切换：选中态左侧加 4px 蓝色边条 + 浅蓝底色 + ring-4 蓝晕
- 流程连接线：步骤之间用 6px 高 的 蓝-靛 垂直渐变线，强化「自上而下顺序执行」的心智

## 布局与响应式

- 桌面优先（≥1280px）：三栏布局
  - 左侧 SOP 库导航：260px 固定
  - 中部画布：流式自适应，max-w-3xl 居中
  - 右侧属性面板：320px 固定
- 顶栏：64px 高，sticky；底部 AI 浮窗右下角，宽 384px / 高 500px
- 移动端非重点（专业工作台场景多在大屏使用），<1024px 时简单折叠右侧面板

## 设计禁忌

- 不要使用大面积渐变铺底——本系统是「工作台」不是「营销页」
- 不要给步骤卡片用霓虹色边框 / 玻璃拟态——破坏专业感
- 不要把判断节点和操作节点设计成同一色系——流程节奏会丢失
- AI Agent 不要做成全屏弹窗——遮挡画布会打断 SOP 编辑思路，必须用悬浮窗
- 不要给主操作按钮加发光阴影——会显得 demo 感
- 不要把表格 / 列表用斑马纹——SOP 卡片本身就有边界，再加斑马纹会噪
